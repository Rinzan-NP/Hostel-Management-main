📋 Project Overview

Hostel Management System is a comprehensive web application designed to streamline hostel operations and management.

  • Purpose: Digitalize and automate hostel administration
  • Scope: Student management, room allocation, and rent tracking
  • Target Users: Hostel administrators and management staff
  • Deployment: Full-stack web application (Cloud-ready)
  • Latest Version: 1.0 with core functionality
✨ Key Features

🏛️ Hall Management

Create and manage multiple hostel halls with detailed location and description information.

🛏️ Room & Bed Management

Track AC/Non-AC rooms, bed capacity, and availability status in real-time.

👥 Student Management

Maintain comprehensive student database with contact info, roll numbers, and department details.

💰 Rent Management

Automated rent tracking, payment status monitoring, and overdue management system.

📊 Dashboard Analytics

Visual insights into occupancy rates, revenue, and hostel statistics.

🔐 User Authentication

Secure login system for hostel administrators and staff members.

🛠️ Technical Stack

Backend

• Framework: Django 6.0.3

• API: Django REST Framework

• Database: SQLite/PostgreSQL

• CORS: django-cors-headers

• Python: 3.8+

Frontend

• Framework: React 19.0

• Build Tool: Vite 5.4

• HTTP Client: Axios

• UI Icons: Lucide React

• Animations: Framer Motion

🏗️ System Architecture

Multi-Tier Architecture

Presentation Layer (Frontend)

React SPA with Vite • Responsive UI • Lucide Icons • Framer Animations

Application Layer (Backend)

Django REST APIs • 6 ViewSets • Serializers • URL Routing

Data Layer

SQLite/PostgreSQL • 6 Models • Relations • Migrations

🗄️ Database Schema
📦 Hall - Hostel halls/buildings
├─ name, location, description
🛏️ Room - Physical rooms
├─ room_number, capacity, type, rent_price, hall_id
🛐 Bed - Individual beds
├─ bed_number, is_occupied, room_id
👤 Student - Student information
├─ name, email, phone, roll_number, department, joined_at
🔑 RoomAssignment - Student-Bed mapping
├─ student_id, bed_id, check_in_date, check_out_date
💵 Rent - Rent payments
├─ assignment_id, amount, due_date, status, paid_at
🔄 Key Workflows
1. Student Admission Workflow
Create Student → Assign Bed → Generate Initial Rent Record → Mark Bed as Occupied
2. Room Assignment Process
Check Hall Availability → Select Room → Verify Bed Capacity → Create Assignment
3. Rent Management Cycle
Generate Monthly Rent → Send Payment Reminder → Mark as Paid/Overdue → Generate Reports
4. Student Checkout
Verify Payment Status → Clear Outstanding Dues → Update Assignment → Mark Bed as Free

🔌 Backend API Endpoints

RESTful API with 6 ViewSets:

  • /api/halls/ - Hall management (CRUD operations)
  • /api/rooms/ - Room management (CRUD operations)
  • /api/beds/ - Bed management (CRUD operations)
  • /api/students/ - Student registry (CRUD operations)
  • /api/room-assignments/ - Student room allocations
  • /api/rents/ - Rent payment tracking and status

Communication Method: JSON over HTTP/HTTPS

API Features: Filtering, Pagination, Search, Sorting

📈 Data Flow & Integration
User Action → Frontend (React) → API Request (Axios)
↓
Backend Server (Django) → Route to ViewSet → Process Logic
↓
Database Operations → Query Execution → Serialization
↓
JSON Response → Frontend Processing → UI Update (Framer Motion)

CORS Enabled: django-cors-headers allows cross-origin requests from the frontend to the backend API.

🚀 Deployment & Setup

Backend Setup:

  • Install dependencies: pip install -r requirements.txt
  • Run migrations: python manage.py migrate
  • Start server: python manage.py runserver
  • Server runs on: http://localhost:8000

Frontend Setup:

  • Install dependencies: npm install
  • Start dev server: npm run dev
  • Frontend runs on: http://localhost:5173

✓ Separate frontend and backend servers ✓ CORS configured for local development
